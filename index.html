Skill-Based Freelancing

EdLance is a dedicated freelancing ecosystem designed exclusively for college students, helping them earn money, build industry-ready skills, and gain real project experience while still pursuing their education. Built with a vision to bridge the gap between academic learning and real-world opportunities, EdLance ensures that every student—regardless of background—can showcase their talent and access meaningful work.

Business System

Java Demo

Small Business Operations

A desktop application for a small business built using Java Swing for the user interface and a MySQL database for the backend. Its main purpose is to centralize and manage all the core operations of a business in one place, replacing manual spreadsheets or paper files.

Key Features

  • Technology: It uses Java Swing (for the GUI), JDBC to connect to a MySQL database, and follows a 3-Tier (DAO) architecture to keep the code clean.
  • Three Core Modules:
    • Inventory Management: Lets you add, update, delete, and search for products. It has a "Check Status" button that shows you which products are low on stock, overstocked, or expired.
    • Employee Management: Lets you add, update, delete, and search for employees. It also has features to log daily attendance and automatically generate a payslip with bonuses and deductions.
    • Financial Management: Shows a dashboard with your total sales, expenses, and profit. You can add new daily financial records and generate a simple monthly report.
  • User-Friendly: All data entry (like adding a new product or employee) is done through a single, clean pop-up window instead of multiple annoying ones.
  • Database Control: It has a "Reset Database" button on the main menu to easily wipe and restart the data, which is useful for testing.

FRISP OS

Systems Engineering

Experimental Microkernel Project

FRISP OS is a research-oriented microkernel operating system built from scratch as a long-term systems engineering project under FryLabs Studio. The project is focused on understanding operating system architecture from first principles — starting at the boot process and progressing toward user-space execution.

The goal of FRISP OS is not commercial release, but deep technical exploration of how modern systems operate at the lowest level.

Core Focus Areas

  • Boot Process & Bare-Metal Initialization: Implementing and validating the boot flow, setting up Multiboot compliance, and preparing the execution environment for kernel entry.
  • Memory Management & Paging: Designing virtual memory structures, enabling paging, and implementing strict memory isolation between kernel and user space.
  • Interrupt Handling & CPU Structures: Setting up GDT and IDT, handling hardware interrupts, and preparing the foundation for stable kernel execution.
  • Microkernel Architecture: Keeping the kernel minimal by isolating essential components (scheduling, memory management, IPC) while moving drivers and services toward user space.
  • Preemptive Multitasking & IPC: Exploring process scheduling, context switching, and secure message-based inter-process communication.

Technologies Used

  • C / C++
  • x86_64 Assembly
  • Bare Metal Development
  • Custom Bootloader & Toolchain

Vision

FRISP OS exists as a hands-on exploration of low-level systems design. The project is driven by curiosity and a desire to understand how operating systems function beneath abstraction layers.

Built progressively, one subsystem at a time.
